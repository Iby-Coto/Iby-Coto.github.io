Birthday Paradox project

This project is all about illustrating the famous “birthday paradox. How likely it is that a group of people will share the same birthday. When you run the simulator, it tries out different group sizes (from 5 up to 100) and, for each size, runs 20 separate trials. Each trial creates random birthdays for everyone in that group and checks if any two people have the same date. The simulator gathers all the results, calculates how often a shared birthday actually appears, and prints a summary. You will see how, even with surprisingly small groups, there is a pretty good chance that at least two people share a birthday.

I collaborated on this project with a classmate, I mainly worked on implementing the Trial class and creating test cases. We adopted a lightweight approach to test-driven development (TDD) where I would write a few preliminary tests to confirm that everything from the constructor’s initialization to the detection of duplicate birthdays behaved correctly, including edge scenarios like handling zero people. This process helped catch small errors early and ensured the Trial class was solid before integrating it. We also did occasional pair programming, where one person wrote a quick test while the other tackled the implementation, making troubleshooting faster and more efficient. Working closely together taught me the importance of clear communication and teamwork. This was my first real introduction to using test cases, and it boosted both my confidence and my ability to ensure code reliability from the outset. It was also my first time forking, creating branches, creating pull request and working with others on GitHub.

Overall, this project was both enjoyable and educational, helping me sharpen my Java coding skills and solidifying the value of collaboration in producing robust, well-tested software.
